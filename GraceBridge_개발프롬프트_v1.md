# [Claude Code 개발 지시서] 기독교(개신교) 목회 지원 플랫폼 — "GraceBridge" (가칭)

> 이 문서 전체를 Claude Code에 붙여넣어 개발을 지시하는 프롬프트입니다.
> 프로젝트명은 가칭이며, 관리자와 협의하여 변경 가능합니다.

---

## 0. 역할 및 기본 원칙

너는 시니어 풀스택 개발자다. 아래 사양에 따라 **완성도 높은 프로덕션급 웹 플랫폼**을 구축하라.

- 임의로 기능을 축소하거나 생략하지 말 것. 불명확한 부분은 구현 전에 질문할 것.
- 각 Phase 완료 시마다 빌드(`npm run build`)가 통과하는 상태를 유지할 것.
- 모든 커밋은 의미 단위로 나누고, 커밋 메시지는 영어로 작성할 것.
- 신학적 콘텐츠(설교, 성경 해설)는 개신교 복음주의 주류 관점을 기본으로 하되, 특정 교단을 배타적으로 옹호하거나 타 교단을 폄하하는 표현을 생성하지 않도록 AI 프롬프트를 설계할 것.

## 1. 프로젝트 개요

| 항목 | 내용 |
|---|---|
| 목적 | 목회자의 설교(강론) 준비를 돕고, 일반 신자의 성경 이해를 돕는 구독형 플랫폼 |
| 주 사용자 | ① 목회자(주 구독자) ② 일반 신자 ③ 관리자(이성규 목사) |
| 핵심 기능 | AI 설교 생성기, 성경 이해 콘텐츠 라이브러리(슬라이드·동영상), AI 슬라이드 생성기(관리자용) |
| 언어 | 한국어(기본)·영어·프랑스어·스페인어·독일어 — UI와 AI 생성 콘텐츠 모두 5개 언어 지원 |
| 대상 시장 | 전 세계 (다국어·다시간대 고려) |
| 배포 | GitHub → Vercel (자동 배포), 모바일 우선(mobile-first) 반응형 |

## 2. 기술 스택 (지정)

- **Framework**: Next.js 14+ (App Router, TypeScript)
- **스타일**: Tailwind CSS, mobile-first 반응형. 시스템 폰트 우선 + 한국어 Noto Sans KR / Pretendard
- **다국어(i18n)**: `next-intl` — URL 경로 기반 로케일(`/ko`, `/en`, `/fr`, `/es`, `/de`), 브라우저 언어 자동 감지 + 수동 전환
- **DB/인증/스토리지**: Supabase (Postgres + Auth + Storage). 이메일/비밀번호 + Google OAuth 로그인
- **AI**: Anthropic API (`claude-sonnet-4-6`). API 키는 서버 사이드(Route Handler)에서만 사용, 클라이언트 노출 금지
- **결제(구독)**: Stripe (해외) 기반으로 설계하되, Phase 1에서는 결제 없이 "무료 체험 + 관리자 수동 승인" 구조로 시작. Stripe 연동 코드는 스캐폴딩만 준비
- **동영상**: YouTube 링크 임베드(unlisted 권장). 자체 업로드는 Supabase Storage(용량 한계 안내 문구 포함)
- **배포**: Vercel. 환경변수: `ANTHROPIC_API_KEY`, `NEXT_PUBLIC_SUPABASE_URL`, `NEXT_PUBLIC_SUPABASE_ANON_KEY`, `SUPABASE_SERVICE_ROLE_KEY`

## 3. 사용자 역할 및 권한

| 역할 | 권한 |
|---|---|
| `admin` (이성규 목사) | 전체 관리: 콘텐츠 업로드/승인/삭제, 회원 관리, 구독 승인, 통계 조회, AI 슬라이드 생성·게시 |
| `pastor` (목회자 구독자) | 설교 생성기 사용(무제한 또는 월 한도), 전체 콘텐츠 열람, 내 설교 보관함 |
| `member` (일반 신자) | 성경 이해 콘텐츠 열람, 묵상 도우미 사용. 설교 생성기는 요약 모드만 |
| `guest` (비로그인) | 랜딩 페이지, 샘플 콘텐츠 3개 미리보기 |

- 회원가입 시 역할 선택(목회자/일반 신자) → 목회자는 소속 교회명·직분 입력 → 관리자 승인 후 `pastor` 권한 부여 (승인 전에는 `member` 취급)
- Supabase RLS(Row Level Security)를 모든 테이블에 적용할 것.

## 4. 핵심 기능 상세

### 4-1. 설교(강론) 생성 탭 — 플랫폼의 핵심

**입력 폼 (구조화된 단계형 UI, 모바일에서 한 화면 한 단계):**

1. **시기**: 절기 선택(대림절/성탄절/사순절/부활절/오순절/추수감사절/송구영신/신년/일반 주일 등) + 자유 입력
2. **장소/예배 유형**: 주일 대예배/수요예배/새벽기도/청년부/장년부/구역예배/장례식/결혼식/헌신예배/야외예배 등
3. **대상 회중**: 연령대(아동/청소년/청년/장년/노년/전 연령), 규모, 신앙 성숙도(초신자 다수/기존 신자 중심/혼합)
4. **분량**: 10분/20분/30분/40분 (분량별 예상 단어 수 자동 계산: 한국어 기준 분당 약 250~300자)
5. **성경 구절**: 책·장·절 선택 UI(드롭다운) + 자유 입력. 복수 구절 허용
6. **설교 스타일**: 강해설교/주제설교/본문설교/이야기식 설교 중 선택
7. **추가 포함 요망사항**: 자유 텍스트 (예: "교회 창립 30주년 언급", "최근 지역 재난 위로 포함")
8. **출력 언어**: 5개 언어 중 선택 (UI 언어와 별개로 설정 가능 — 예: 한국어 UI에서 영어 설교 생성)

**생성 결과 구성 (필수 섹션):**

- 설교 제목 (3개 후보 제시)
- 본문 개요 (서론–본론 2~3대지–결론)
- 전체 설교문 (선택 분량에 맞춤)
- **실제 사례(예화) 최소 2개 이상 포함** — 역사적 사례, 일상 사례, 현대적 적용 사례. 단, 실존 인물 예화는 검증 가능한 널리 알려진 사실만 사용하고, 출처가 불확실한 예화는 "가상의 예화"임을 명시하도록 AI 프롬프트에 규칙을 넣을 것
- 적용 질문 3개 (소그룹 나눔용)
- 대표 기도문 (선택 사항 체크박스)
- 회중용 요약 유인물 (A5 1장 분량)

**부가 기능:**

- 생성 결과 편집기(인라인 수정 가능) → "내 설교 보관함"에 저장
- DOCX / PDF / 일반 텍스트 다운로드
- 재생성 버튼("예화를 다른 것으로", "더 따뜻한 톤으로", "더 간결하게" 등 원클릭 수정 요청)
- 설교 생성 이력 관리, 검색
- **면책 문구 고정 표시**: "본 초안은 설교 준비를 돕는 참고 자료입니다. 최종 설교는 목회자의 기도와 묵상, 신학적 검토를 거쳐 완성하시기 바랍니다." (5개 언어)

**AI 시스템 프롬프트 설계 규칙 (서버 측):**

- 성경 인용은 요청 언어의 표준 역본 기준(한국어: 개역개정 / 영어: NIV 또는 ESV / 프랑스어: Louis Segond / 스페인어: Reina-Valera / 독일어: Luther Bibel)을 **표기**하되, 저작권 문제를 피하기 위해 긴 구절은 장·절 참조 중심으로 안내하고 직접 인용은 짧게 제한
- 이단적 해석, 번영신학 극단, 정치적 선동을 배제하는 가드레일 명시
- 스트리밍 응답(SSE)으로 생성 과정을 실시간 표시 (긴 생성 시간 체감 완화)

### 4-2. 성경 이해 콘텐츠 라이브러리 (슬라이드·동영상)

- 성경 66권 구조의 브라우징 UI: 구약/신약 → 책 → 콘텐츠 목록
- 콘텐츠 유형: ① 슬라이드(플랫폼 내 뷰어) ② 동영상(YouTube 임베드) ③ PDF 자료
- 각 콘텐츠: 제목, 설명, 대상(목회자용/전체), 언어 태그, 조회수, 북마크
- 검색(제목·책·주제) + 필터(언어/유형/대상)
- 모바일에서 슬라이드는 스와이프로 넘기는 전체 화면 뷰어 제공

### 4-3. AI 슬라이드 생성기 (관리자 도구 — 플랫폼 내 제작 지원)

관리자가 외부 도구 없이 플랫폼 안에서 슬라이드를 만들 수 있도록 한다:

1. 성경 구절/주제 + 대상 + 슬라이드 매수(5~20장) 입력
2. AI가 슬라이드 구성안(각 장의 제목·본문·시각 요소 제안)을 JSON으로 생성
3. HTML 기반 슬라이드로 즉시 렌더링(16:9, 통일된 디자인 테마 3종 중 선택)
4. 관리자가 장별로 텍스트 편집 → 미리보기 → **게시** 버튼으로 라이브러리에 등록
5. PDF 내보내기 지원 (인쇄·배포용)

동영상은 플랫폼 내 자동 제작 대신, 관리자가 제작한 영상의 YouTube URL을 등록하는 방식으로 처리한다. (관리자 페이지에 "영상 등록" 폼: URL 붙여넣기 → 썸네일·제목 자동 추출)

### 4-4. 일반 신자용 기능

- **오늘의 묵상**: 매일 성경 구절 + AI 생성 짧은 묵상글(관리자 사전 승인 큐 방식 — AI가 7일치 미리 생성 → 관리자 승인 → 게시)
- **성경 질문하기**: 성경 내용에 대한 Q&A 챗봇 (신학적 논쟁 주제는 "목회자와 상담을 권합니다" 응답으로 우회)
- 콘텐츠 북마크 / 읽음 표시

### 4-5. 관리자 대시보드 (`/admin` — admin 권한만 접근)

- 회원 목록·역할 관리·목회자 승인 대기열
- 콘텐츠 관리(업로드·수정·게시·비공개·삭제)
- AI 슬라이드 생성기(4-3)
- 묵상글 승인 큐
- 사용 통계: 가입자 추이, 언어별 사용자 분포, 설교 생성 횟수, 인기 콘텐츠
- 공지사항 작성(5개 언어 — 한국어 작성 후 "AI 번역" 버튼으로 4개 언어 초안 자동 생성 → 관리자 확인 후 게시)

## 5. 데이터베이스 스키마 (Supabase)

```
profiles        (id, email, display_name, role, church_name, position, locale, approved, created_at)
sermons         (id, user_id, title, inputs_json, content_md, language, created_at, updated_at)
contents        (id, type[slide|video|pdf], title, description, book, language, audience,
                 slide_json, video_url, file_url, published, view_count, created_by, created_at)
bookmarks       (id, user_id, content_id, created_at)
devotionals     (id, date, verse_ref, body_md, language, status[draft|approved|published], created_at)
announcements   (id, title, body_md, language, published_at)
usage_logs      (id, user_id, action, meta_json, created_at)
```

모든 테이블에 RLS 정책 작성. `sermons`는 본인만 조회/수정, `contents`는 published=true만 일반 조회.

## 6. 다국어(i18n) 구현 규칙

- UI 문자열은 전부 `messages/{ko,en,fr,es,de}.json`으로 분리. 하드코딩 금지
- 한국어를 원본으로 작성 후 4개 언어 번역 파일 생성 (번역 품질은 자연스러운 현지 표현 우선)
- 날짜/숫자 형식 로케일 대응 (`Intl` API)
- 성경 책 이름은 언어별 표준 명칭 매핑 테이블 작성 (예: 창세기/Genesis/Genèse/Génesis/1. Mose)
- SEO: 언어별 `<html lang>`, hreflang 태그, 언어별 메타데이터

## 7. 모바일 최적화 규칙

- 뷰포트 375px 기준 우선 설계, 데스크톱은 확장 레이아웃
- 하단 탭 바 내비게이션(모바일): 홈 / 설교 생성 / 라이브러리 / 묵상 / 내 정보
- 설교 생성 폼은 단계형(step wizard), 한 화면에 한 질문
- 터치 타깃 최소 44px, 폰트 최소 16px(iOS 자동 확대 방지)
- 긴 설교문 읽기 모드: 글자 크기 조절, 다크 모드 지원

## 8. 보안·품질 요구사항

- Anthropic API 호출은 반드시 서버(Route Handler)에서. 사용자별 rate limit(예: 목회자 1일 10회 생성, member 1일 3회 요약)
- 입력값 검증(zod), XSS 방지(생성 마크다운은 sanitize 후 렌더)
- 관리자 페이지는 미들웨어에서 role 검사
- 에러 바운더리 + 사용자 친화적 에러 메시지(5개 언어)
- Lighthouse 모바일 성능 80점 이상 목표

## 9. 개발 단계 (Phase)

**Phase 1 — 기반 구축**
프로젝트 스캐폴딩, i18n 구조, Supabase 연동, 인증(가입/로그인/역할), 레이아웃(모바일 탭 바 포함), 랜딩 페이지

**Phase 2 — 설교 생성기 (핵심)**
입력 폼 위저드, AI 생성 API(스트리밍), 결과 뷰어/편집기, 보관함, DOCX/PDF 내보내기, rate limit

**Phase 3 — 콘텐츠 라이브러리**
66권 브라우징, 슬라이드 뷰어, YouTube 임베드, 검색/필터, 북마크

**Phase 4 — 관리자 도구**
관리자 대시보드, 회원/승인 관리, 콘텐츠 CRUD, AI 슬라이드 생성기, 묵상 승인 큐, 통계

**Phase 5 — 신자 기능 + 마감**
오늘의 묵상, 성경 Q&A, 공지, 다크 모드, SEO, 성능 최적화, Vercel 배포 설정(`vercel.json`, 환경변수 문서), README(설치·배포·관리자 매뉴얼 포함, 한국어)

각 Phase 완료 시: 빌드 통과 확인 → 주요 화면 스크린샷 또는 요약 보고 → 다음 Phase 진행 여부 확인 요청.
